◎ 제출물 : 하드카피(프로젝트 평가표, 프로그램 개요, 프로그램 소스, 함수 설명, 실행 화면, 참여자들의 작업 분배 항목과 참여 비율 명확히 명시), 프로그램 소스 파일.
          하드카피는 실습시간에 제출하고 소스 파일은 조교에게 이메일 제출 

◎ 기한 : 12월 8~12일 실습시간에 제출
◎ 3인 혹은 4인으로 팀을 구성해야 하며 그렇지 않을 경우 50%만 부여
◎ 카피본 발견 시 소스를 제공한 팀과 함께 0점 처리
◎ 고스톱 게임 프로그램 요구 조건 :

0. 모든 화투 패는 동적 메모리 할당 기법과 자기 참조 구조체로 관리한다. (든 패, 먹은 패, 깔린 패, 화투 더미)
1. 3 명(A, B, C)이 고스톱을 할 수 있는 환경을 제공해야 한다.
2. 첫 판은 A가 선을 하고, 그 다음부터는 이전 판을 이긴 사람이 선을 한다.
3. 화면에 3 명의 패와 각 사람이 먹은 패, 그리고 깔린 패를 보여주고, 현재 누가 할 차례인지를 표시해야 한다.
- 단, 든 패와 먹은 패, 깔린 패는 화투의 번호로 정렬되어야 한다.
4. 흔들기, 설사, 판 쓰리, 고, 총통, 나가리 등을 처리해야 한다.
- 단, 흔들기는 패를 돌렸을 때 자동으로 흔들도록 한다.
- 고를 한 후에는 다음에 점수가 추가되어야만 고나 스톱을 할 수 있다.
- 막판에는 고를 못한다.
5. 한 번에 같은 무늬를 4장 먹으면 다른 사람은 피를 한 장씩 준다.
- 싼 것 먹기, 따닥이 이에 해당된다.
- 쌍 피만 있으면 쌍 피를 준다.
- 막 판에도 유효하다.
6. 점수는 다음과 같이 계산된다.
* 광 점수
- 광을 3장 이상 모으면 점수가 되고 다음 규칙이 적용된다.
1) 비 광을 제외한 광이 3장이면 3점
2) 비 광을 포함하여 광이 3장이면 2점
3) 광이 4장이면 4점
4) 광이 5장이면 15점
* 10 자리 점수
1) 10 자리를 5장 모으면 1점이고, 한 장씩 추가될 때마다 1점씩 추가가 됨
2) 고도리는 5점
* 5 자리로 점수
1) 5 자리를 5장 모으면 1점이고, 한 장씩 추가될 때마다 1점씩 추가가 됨
2) 청단, 초단, 홍단 : 각 3점씩
* 피로 점수
1) 피를 10장 모으면 1점이고, 한 장씩 추가될 때마다 1점씩 추가가 됨
** 쌍피는 피 2 장으로 계산해야 함
* 총통
1) 같은 무늬의 패가 4개 모두 들어오면 10점으로 계산하며 게임이 끝남
7. 돈 계산
1) 각 사람은 10만원으로 시작하고, 판이 끝날 때마다 잔고 계산을 해야 한다.
2) 각 판이 끝나면 피박, 고박, 쓰리고, 포고, 흔들기, 멍텅구리 등을 고려하여 돈 계산을 한다. 이때 점 당 100원으로 하고, 여러 경우가 복합된 경우는 계속 2배씩 돈 계산을 한다. 
   - 예) 쓰리고를 했는데 피박이면 4배.
         흔든 사람이 쓰리 고를 했는데 피박이면 8배.
8. 사용자 인터페이스
* 사용자는 키보드를 이용하여 게임을 할 수 있어야 하고, 게임 환경은 다음 키들을 처리해야 한다. 

1) g(o) : 고 (자기 turn에 점수가 났고 3점 이상일 때 할 수 있음, )
2) s(top) : 스톱 (자기 turn에 점수가 났고 3점 이상일 때 할 수 있음, 이번 판을 끝냄)
3) e(xit) : exit (프로그램 끝내기) 
4) b(alance) : 게이머의 잔고 보기
5) h(elp) : 각 키에 대한 설명 보기
6) 1~7 : 낼 화투 선택
7) 1~2 : 화투를 냈는데, 깔린 화투 중 무늬는 같지만 값이 다른 것이 있을 때 선택
8) 9 : 9 십을 피로 또는 십으로 이동 (토글), 디폴트로는 피로 함, 각 판에서 한번만 할 수 있음.
9) save : 현재 상태를 파일에 저장 (단, 확인 가능하도록 텍스트 형태로 저장해야 함)
10) load : 파일에 저장된 상태를 읽어서 계속 게임 진행
